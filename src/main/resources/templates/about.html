Documentación Técnica

DAC: Digital Access Control

Una solución de grado empresarial diseñada para centralizar, proteger y gestionar las credenciales críticas del personal de Claro con estándares internacionales de cifrado.

The Stack

  • Frontend: HTML5 / Tailwind CSS 3
  • Interacción: JavaScript ES6+
  • Validación: Constraint API (Regex)
  • Diseño: Mobile First & Responsive

Seguridad

  • Cifrado: AES-256-GCM
  • Hashing: Argon2id / Bcrypt
  • Protección: TLS 1.3 In-Transit
  • Identidad: Validación de Cédula RD

Base de Datos

  • Motor: Relacional (PostgreSQL)
  • Integridad: Llaves Foráneas ST
  • Vault: Estructura BLOB Encriptada
  • Auditoría: Logs de acceso (Logs)

Detalles Técnicos de Encriptación

Algoritmo AES-256-GCM

Utilizamos el estándar AES de 256 bits en modo GCM. A diferencia de otros modos, GCM proporciona autenticación de datos, lo que significa que si alguien intenta modificar el dato encriptado en la base de datos, el sistema lo detectará inmediatamente al intentar descifrarlo.

Arquitectura de Base de Datos

Los datos del usuario (Cédula, Empleado, Email) están separados lógicamente de la "Bóveda". Cada entrada en la bóveda requiere una derivación de llave única que solo existe mientras la sesión del usuario está activa.

Ventajas del Sistema DAC

Agilidad Corporativa

Acceso inmediato a herramientas internas sin depender de recuperaciones de contraseña constantes.

Cumplimiento (Compliance)

Sigue estrictamente las políticas de seguridad de la información de Claro Dominicana.
